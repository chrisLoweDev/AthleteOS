# Plan: /push-workouts — Hevy + MyWhoosh Integration

## Context
The user wants planned workout files (generated by `/plan-workouts`) to be automatically pushed to two external apps:
- **Hevy** — receives weight training sessions as **Routines** (templates the user can start in-app)
- **MyWhoosh** — receives cycling sessions as **custom workouts** (structured power-based sessions)

This is triggered by a new `/push-workouts` slash command, run manually after planning.

---

## What Gets Built

### 1. `.env` additions
Add three new credential fields (blank values, user fills in):
```
HEVY_API_KEY=
MYWHOOSH_EMAIL=
MYWHOOSH_PASSWORD=
```

### 2. Workout frontmatter additions
Weights files get: `hevy_routine_id: null`
Cycling files get: `mywhoosh_workout_id: null`

These prevent duplicate pushes on re-runs.

Update the 5 existing W11 plan files (and document in CLAUDE.md template).

### 3. `scripts/push_hevy.py`
**Input:** path to a weights markdown file, reads `HEVY_API_KEY` from `.env`

**Logic:**
1. Parse YAML frontmatter; skip if `hevy_routine_id` already set
2. Parse the **Main Lifts** and **Accessory Work** markdown tables using regex
3. Group rows by exercise name; determine set type from Notes column ("Warm-up" → `"warmup"`, else `"normal"`)
4. For each unique exercise name: `GET https://api.hevyapp.com/v1/exercise_templates?searchValue=<name>&pageSize=5` — pick first result's `id`
5. Build POST body: `{ "routine": { "title": "<key_focus>", "exercises": [...] } }`
   - Each exercise: `exercise_template_id`, `notes`, `sets: [{ type, weight_kg, reps }]`
6. `POST https://api.hevyapp.com/v1/routines` with header `api-key: <HEVY_API_KEY>`
7. Print returned `routine.id` to stdout; errors to stderr

**Exercise name → Hevy search term mapping** (hardcoded dict):
```python
HEVY_NAMES = {
    "Squat (Barbell)": "Barbell Squat",
    "Bench Press (Barbell)": "Bench Press (Barbell)",
    "Bent-Over Row (Barbell)": "Bent Over Row (Barbell)",
    "Romanian Deadlift (Barbell)": "Romanian Deadlift (Barbell)",
    "Overhead Press (Barbell)": "Overhead Press (Barbell)",
    "Pull-Ups": "Pull Up",
    "Hanging Leg Raises": "Hanging Leg Raise",
    "Bicep Curl (Barbell)": "Barbell Curl",
    "Russian Twists": "Russian Twist",
}
```

**Auth:** Header `api-key: <HEVY_API_KEY>`. If key is empty, print instructions and exit with code 2 (distinct from other errors).

### 4. `scripts/push_mywhoosh.py`
**Input:** path to a cycling markdown file, FTP value (from profile), reads `MYWHOOSH_EMAIL` + `MYWHOOSH_PASSWORD` from `.env`

**Logic:**
1. Parse YAML frontmatter; skip if `mywhoosh_workout_id` already set
2. Parse section headers + content:
   - `## Warm Up (N min)` → E_WarmUp step, ramp 0.40→0.55 FTP, duration N×60s
   - `## Main Set (N min)` → E_Normal step, parse watts from `**W1–W2W**` pattern → use midpoint / FTP, duration N×60s
   - `## Cool Down (N min)` → E_CoolDown step, ramp 0.55→0.40 FTP, duration N×60s
3. Login: `POST https://services.mywhoosh.com/http-service/api/login` with `{"Username": email, "Password": password, "Platform": "Android", "Action": 1001, "CorrelationId": str(uuid4()), "DeviceId": "athleteos", "Authorization": ""}`
   → extract `AccessToken` and `WhooshId`
4. Build workout payload:
   ```json
   {
     "UserId": "<WhooshId>",
     "WorkoutsData": [{
       "Id": "<uuid4>",
       "Name": "<key_focus>",
       "Description": "Generated by Athlete OS",
       "Mode": "E_Ride",
       "Time": <total_seconds>,
       "Type": "E_Custom",
       "DisplayType": "E_byWatts",
       "StepCount": <len(steps)>,
       "WorkoutStepsArray": [...]
     }]
   }
   ```
5. `POST https://coaching.mywhoosh.com/api/v2/client/custom-workout-upload` with `Authorization: Bearer <AccessToken>`
6. Print the workout `Id` to stdout; errors to stderr

**FTP argument:** passed as `--ftp 270` CLI flag; script reads from flag, not profile directly (keeps it standalone like the other scripts).

### 5. `.claude/commands/push-workouts.md`
New slash command. Steps:
1. Read `athlete/profile.md` for FTP
2. Glob `workouts/plans/**/*.md` — find all pending files
3. Split into weights files (type: weights) and cycling files (type: cycling)
4. Check `.env`: warn if `HEVY_API_KEY` is empty (but proceed for MyWhoosh); warn if `MYWHOOSH_EMAIL`/`MYWHOOSH_PASSWORD` empty (but proceed for Hevy)
5. For each weights file where `hevy_routine_id: null`:
   - Run `python3 scripts/push_hevy.py <file_path>`
   - On success: edit frontmatter `hevy_routine_id: <id>`
   - On exit code 2 (no API key): explain how to get Hevy Pro key, skip remaining
   - On other error: display stderr, continue with remaining files
6. For each cycling file where `mywhoosh_workout_id: null`:
   - Run `python3 scripts/push_mywhoosh.py <file_path> --ftp <FTP>`
   - On success: edit frontmatter `mywhoosh_workout_id: <id>`
   - On error: display stderr, continue
7. Print summary table of results

## Key API Details (for reference during implementation)

**Hevy:**
- Base URL: `https://api.hevyapp.com/v1`
- Auth header: `api-key: <key>`
- Create routine: `POST /routines` body `{ "routine": { "title", "exercises": [{ "exercise_template_id", "notes", "sets": [{ "type", "weight_kg", "reps" }] }] } }`
- Lookup exercises: `GET /exercise_templates?searchValue=<name>&pageSize=5`
- Set types: `"warmup"` | `"normal"` | `"failure"` | `"dropset"`

**MyWhoosh:**
- Login: `POST https://services.mywhoosh.com/http-service/api/login`
- Upload: `POST https://coaching.mywhoosh.com/api/v2/client/custom-workout-upload`
- Auth: `Authorization: Bearer <AccessToken>`
- Power values: FTP fraction (e.g., 0.65 = 65% FTP)
- Step types: `E_WarmUp` (ramp: StartPower→EndPower), `E_Normal` (flat Power), `E_CoolDown` (ramp), `E_FreeRide`
