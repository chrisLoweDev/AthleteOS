# Plan: /push-workouts — Hevy Integration

## Context
The user wants planned workout files (generated by `/plan-workouts`) to be automatically pushed to Hevy as **Routines** (templates the user can start in-app).

This is triggered by a new `/push-workouts` slash command, run manually after planning.

---

## What Gets Built

### 1. `.env` additions
Add one new credential field (blank value, user fills in):
```
HEVY_API_KEY=
```

### 2. Workout frontmatter additions
Weights files get: `hevy_routine_id: null`

This prevents duplicate pushes on re-runs.

Update the 2 existing W11 weights plan files (and document in CLAUDE.md template).

### 3. `scripts/push_hevy.py`
**Input:** path to a weights markdown file, reads `HEVY_API_KEY` from `.env`

**Logic:**
1. Parse YAML frontmatter; skip if `hevy_routine_id` already set
2. Parse the **Main Lifts** and **Accessory Work** markdown tables using regex
3. Group rows by exercise name; determine set type from Notes column ("Warm-up" → `"warmup"`, else `"normal"`)
4. Look up each exercise name in `HEVY_EXERCISE_IDS`; if not found, print error with the unmatched name and exit with code 3
5. Build POST body: `{ "routine": { "title": "<key_focus>", "exercises": [...] } }`
   - Each exercise: `exercise_template_id`, `notes`, `sets: [{ type, weight_kg, reps }]`
6. `POST https://api.hevyapp.com/v1/routines` with header `api-key: <HEVY_API_KEY>`
7. Print returned `routine.id` to stdout; errors to stderr

**Exercise name → Hevy template ID mapping** (hardcoded, IDs verified via API on 2026-03-13):
```python
HEVY_EXERCISE_IDS = {
    "Squat (Barbell)":            "D04AC939",  # Squat (Barbell)
    "Bench Press (Barbell)":      "79D0BB3A",  # Bench Press (Barbell)
    "Bent-Over Row (Barbell)":    "55E6546F",  # Bent Over Row (Barbell)
    "Romanian Deadlift (Barbell)":"2B4B7310",  # Romanian Deadlift (Barbell)
    "Overhead Press (Barbell)":   "7B8D84E8",  # Overhead Press (Barbell)
    "Pull-Ups":                   "1B2B1E7C",  # Pull Up
    "Hanging Leg Raises":         "F8356514",  # Hanging Leg Raise
    "Bicep Curl (Barbell)":       "A5AC6449",  # Bicep Curl (Barbell)
    "Russian Twists":             "BB83BDDE",  # Russian Twist (Bodyweight)
}
```

No API lookup at runtime — IDs are stable. If an exercise name isn't in this dict, print a clear error and exit.

**Auth:** Header `api-key: <HEVY_API_KEY>`. If key is empty, print instructions and exit with code 2 (distinct from other errors).

### 4. `scripts/hevy_lookup.py`
**Usage:** `python3 scripts/hevy_lookup.py <search term>`

Maintenance utility for finding Hevy exercise template IDs when adding new exercises to `HEVY_EXERCISE_IDS`.

**Logic:**
1. Read `HEVY_API_KEY` from `.env`; exit with instructions if missing
2. Paginate `GET /v1/exercise_templates` (all pages, pageSize=100)
3. Case-insensitive filter by search term against `title`
4. Print matching rows: `<id>  <title>  (<equipment>, <primary_muscle_group>)`

Never called by `push_hevy.py` — run manually when updating the ID dict.

### 5. `.claude/commands/push-workouts.md`
New slash command. Steps:
1. Glob `workouts/plans/**/*.md` — find all pending weights files (`type: weights`)
2. Check `.env`: warn if `HEVY_API_KEY` is empty and exit with instructions
3. For each weights file where `hevy_routine_id: null`:
   - Run `python3 scripts/push_hevy.py <file_path>`
   - On success: edit frontmatter `hevy_routine_id: <id>`
   - On exit code 2 (no API key): explain how to get Hevy Pro key, stop
   - On exit code 3 (unknown exercise): display the unmatched name, stop
   - On other error: display stderr, continue with remaining files
4. Print summary table of results

## Key API Details

**Hevy:**
- Base URL: `https://api.hevyapp.com/v1`
- Auth header: `api-key: <key>`
- Create routine: `POST /routines` body `{ "routine": { "title", "exercises": [{ "exercise_template_id", "notes", "sets": [{ "type", "weight_kg", "reps" }] }] } }`
- Set types: `"warmup"` | `"normal"` | `"failure"` | `"dropset"`

## Verification

1. No API key: run `/push-workouts` → should print instructions with link to hevy.com/settings?developer, exit cleanly
2. With key: check hevy.com → new routine should appear under "My Routines" with correct exercises and sets
3. Re-run on already-pushed files → should skip (IDs already set in frontmatter), print "already pushed" for each
4. Unknown exercise name → should print a clear error with the unmatched name, not silently fail
